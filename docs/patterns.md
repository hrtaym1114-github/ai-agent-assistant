# 8つの活用パターン

AIエージェントを業務に活用するための8つのパターン。
「こんな悩みありませんか？」→「このパターンで解決できます」の逆引きで使える。

---

## P1: プロジェクト指示書パターン（CLAUDE.md設計）

| 項目 | 内容 |
|------|------|
| **解決する課題** | AIに毎回同じことを説明するのが面倒、プロジェクトのルールが属人化 |
| **概要** | プロジェクトのルール・構造・制約をCLAUDE.mdに一元記述し、AIが常にコンテキストを把握した状態で支援 |
| **実装要素** | CLAUDE.md |
| **難易度** | 低（テキスト記述のみ） |
| **導入時間** | 30分〜1時間 |
| **効果実感** | 即日 |

**含めるべき要素**: プロジェクト概要、フォルダ構成、命名規則、作業フロー、禁止事項、頻出タスク手順

**適用例**:
- 営業チーム: 提案書フォルダ構成、顧客名規則、見積もりフロー
- 人事部門: 採用フォルダ構成、候補者管理フロー
- 開発チーム: コード規約、ブランチ戦略、レビュープロセス

## P2: 常駐アシスタントパターン

| 項目 | 内容 |
|------|------|
| **解決する課題** | 特定スキルを日常業務の中で自然に伸ばしたい |
| **概要** | CLAUDE.mdに「すべての応答に対してX処理を追加する」と記述し、意識せずにスキル向上や品質チェックが走る仕組み |
| **実装要素** | CLAUDE.md（指示セクション） |
| **難易度** | 低 |
| **導入時間** | 15分 |
| **効果実感** | 即日 |

**バリエーション**:
| 名前 | 自動で付加される処理 |
|------|-------------------|
| 英語学習モード | 日本語入力→英訳+ポイント解説 |
| 法務チェックモード | 契約文書→リスクポイント指摘 |
| SEOモード | 記事作成→SEOスコア+改善提案 |
| トーン統一モード | 外部向け文書→ブランドトーン一致確認 |

## P3: 情報トリアージパターン

| 項目 | 内容 |
|------|------|
| **解決する課題** | 情報が散らかる、メモしたけどどこに入れたかわからない |
| **概要** | 雑なインプットをAIが自動で分類・構造化し、適切な場所に振り分ける |
| **実装要素** | CLAUDE.md + Skills（コマンド） |
| **難易度** | 中 |
| **導入時間** | 1〜2時間 |
| **効果実感** | 1週間 |

**適用例**:
- 営業: 商談メモ→CRM登録用データ+次アクション抽出
- 企画: ブレスト結果→アイデア分類+実現可能性タグ付け
- 管理職: 会議メモ→決定事項+TODO+共有事項に自動分離

## P4: パーソナライズ学習パターン

| 項目 | 内容 |
|------|------|
| **解決する課題** | AIの出力が自分っぽくない、毎回トーンを修正する手間 |
| **概要** | ユーザーの過去の出力からスタイルを学習し、以降の生成に反映 |
| **実装要素** | CLAUDE.md + Skills（学習+生成コマンド） |
| **難易度** | 中〜高 |
| **導入時間** | 2〜3時間（初期学習含む） |
| **効果実感** | 3日 |

**学習対象**: 送信メール、過去の提案書、コードレビュー、議事録

## P5: ガバナンス・ガードレールパターン

| 項目 | 内容 |
|------|------|
| **解決する課題** | AIが暴走しないか心配、機密情報漏洩が不安 |
| **概要** | deny/allow設定、作業範囲制限、確認ステップで安全にAIを活用できる環境を構築 |
| **実装要素** | settings.json + CLAUDE.md |
| **難易度** | 低〜中 |
| **導入時間** | 30分〜1時間 |
| **効果実感** | 即日（安心感） |

**3層防御モデル**:
```
Layer 1: 禁止（deny）    — 絶対にやらせない操作を定義
Layer 2: 制限（scope）   — 作業フォルダ・ファイル範囲を限定
Layer 3: 確認（confirm） — 高リスク操作は人間の承認を要求
```

詳細は [governance.md](governance.md) を参照。

## P7: テンプレート駆動パターン

| 項目 | 内容 |
|------|------|

## P8: ルーティン自動化パターン

| 項目 | 内容 |
|------|------|
| **解決する課題** | 毎日同じ作業を手動でやっている、忘れることがある |
| **概要** | 日次・週次・月次の定型作業をSkillsとして定義し、コマンド一発で実行 |
| **実装要素** | Skills（.claude/commands/） |
| **難易度** | 低〜中 |
| **導入時間** | 1〜2時間（ルーティン1種あたり） |
| **効果実感** | 翌日 |

**設計の3要素**: トリガー（いつ）× アクション（何を）× アウトプット（何が出来る）

## 推奨導入順序

```
Week 1: P1（指示書）+ P5（ガバナンス）
  → 「AIが自分のプロジェクトを理解してくれる」体験

Week 2: P7（テンプレート）+ P8（ルーティン）
  → 「毎日の作業が楽になった」体験

Week 3-4: P3（トリアージ）+ P2（常駐）
  → 「AIが勝手に整理してくれる」体験

Month 2+: P4（学習）+ P6（記録）
  → 「AIが自分の仕事のやり方を覚えてくれる」体験
```

## スキル化と定着 — 「一度きり」で終わらせない

構築したパターンは**カスタムコマンド（スキル）**にすることで、日常に定着する。

### カスタムコマンド化のメリット

| メリット | 説明 |
|---------|------|
| **ワンコマンド実行** | `/weekly-report` と打つだけで毎回の作業を自動化 |
| **チーム共有** | `.claude/commands/` を共有すれば全員が使える |
| **成長する仕組み** | 使いながら改善でき、どんどん自分仕様に |
| **属人化の解消** | 誰でも同じクオリティで作業できる |

### パターン別スキル化の例

| パターン | カスタムコマンド例 | 効果 |
|---------|-----------------|------|
| P1（指示書） | — | CLAUDE.mdは常駐するため不要 |
| P3（トリアージ） | `/triage [メモ]` | メモを自動分類・構造化 |
| P4（学習） | `/my-style [文書種別]` | 学習した文体で文書生成 |
| P7（テンプレート） | `/draft-report`, `/draft-email` | テンプレートから即座に生成 |
| P8（ルーティン） | `/morning`, `/daily-end` | ルーティン作業を自動実行 |

### スキル化の手順

```
1. /ai --build P8 でスキルを構築
2. コマンド名を決める（例: weekly-report）
3. .claude/commands/[名前].md が自動生成される
4. 以降は /[名前] で実行するだけ
```

## 成功体験の展開 — 周囲に広める

### なぜ展開が大切か

- 一人の成功体験が、チーム全体の生産性向上につながる
- 「自分にもできそう」と思ってもらうことが最初の一歩
- 具体的な数字（Before/After）が最強の説得材料

### 展開の3ステップ

```
Step 1: 自分で使って成功体験を得る
  → /ai で構築 → カスタムコマンド化 → 日常で使う

Step 2: 成功体験を記録する
  → /ai --share で共有パックを作成

Step 3: 周囲に共有する
  → 共有パックを配布 → 興味を持った人に /ai を案内
```

### Office業務での活用アピール

文書作成・レポート・メール等のOffice業務は、特に効果が伝わりやすい:

- **PDF分析**: 読み取り→要約→レポート化がワンコマンド
- **週報/日報**: 箇条書き入力→フォーマット済み文書が30秒で完成
- **メール作成**: 自分の文体で即座にドラフト生成
- **データ集計**: CSV読み取り→分析→レポート化を自然言語で指示

詳細は [showcase.md](showcase.md) を参照。
