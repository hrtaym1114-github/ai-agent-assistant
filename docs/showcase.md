# Claude Code 実力ショーケース

> Claude Codeが「何ができるか」を実際に見せるためのリファレンス。
> `/ai --demo` やセッション中のライブデモで参照する。

---

## Claude Codeの強み — コード不要の万能ツール

Claude Codeは「コーディングツール」という名前ですが、**コードを書かなくても使えます**。
Anthropic社内でも法務・財務・PMなど非エンジニアが日常業務に活用しています。

---

## できること一覧

### 1. ファイルの読み取り・分析

| 対象 | できること | 具体例 |
|------|----------|--------|
| **PDF** | そのまま読み取り、要約・分析・比較 | 契約書のリスクポイント抽出、論文の要点整理 |
| **画像** | スクリーンショットや図表の内容理解 | 画面キャプチャから手順書を自動生成、グラフの数値読み取り |
| **CSV/TSV** | データ分析、統計、パターン発見 | 売上データの月別推移分析、異常値検出 |
| **Markdown/テキスト** | 構造化、校正、翻訳、要約 | 議事録の要点抽出、英日翻訳 |
| **Jupyter Notebook** | コードと出力の解説、改善提案 | 分析ノートブックのレビュー |

### 2. 文書の生成・編集

| アクション | 具体例 |
|-----------|--------|
| **テンプレートから生成** | 週報・日報・議事録・提案書を構造化して作成 |
| **文体の調整** | フォーマル/カジュアル、社内向け/社外向けの切り替え |
| **多言語対応** | 日本語→英語、英語→日本語の翻訳（業務文脈を理解） |
| **校正・改善** | 誤字脱字チェック、論理構成の改善提案 |
| **要約** | 長文を3行に、あるいは目的に応じた粒度で |

### 3. データの整理・加工

| アクション | 具体例 |
|-----------|--------|
| **CSVデータ加工** | フィルタリング、集計、ピボット、結合 |
| **レポート生成** | データからグラフ用サマリーやレポートを自動作成 |
| **フォーマット変換** | CSV→Markdown表、JSON→CSV、テキスト→構造化データ |
| **バッチ処理** | フォルダ内の全ファイルに同じ処理を適用 |

### 4. 自動化・ワークフロー

| アクション | 具体例 |
|-----------|--------|
| **カスタムコマンド作成** | `/morning`, `/weekly-report` 等のワンコマンド自動化 |
| **フォルダ整理** | ファイルの自動分類、命名規則の適用 |
| **定型作業の自動化** | 毎日/毎週の繰り返し作業をコマンド化 |
| **チェックリスト実行** | 手順書に沿った確認作業の自動化 |

### 5. 検索・ナビゲーション

| アクション | 具体例 |
|-----------|--------|
| **ファイル横断検索** | 「○○に関する資料はどこ？」でプロジェクト横断検索 |
| **コンテキスト理解** | フォルダ構成を理解し、適切な場所にファイルを配置 |
| **関連情報の発見** | 「このファイルに関連する資料を探して」 |

## Office業務との連携

### Claude Code × Microsoft Office ワークフロー

Claude Codeは直接Office形式のファイルを出力しませんが、
**実用的なワークフロー**でOffice業務を大幅に効率化できます。

#### Word/文書作成

```
作業フロー:
1. Claude Codeに「○○について報告書を作って」と依頼
2. Markdown形式で構造化された文書が生成される
3. そのままWordに貼り付け → 書式が自動適用
4. 微調整を指示 →「もっと簡潔に」「箇条書きに変えて」
```

#### Excel/データ分析

```
作業フロー:
1. CSVファイルをClaude Codeに読ませる
2. 「月別の売上推移を集計して」と自然言語で指示
3. 集計結果がCSVまたはMarkdown表で出力
4. Excelに貼り付け → グラフ化
```

#### PowerPoint/プレゼン

```
作業フロー:
1. Claude Codeに「○○のプレゼン構成を考えて」と依頼
2. スライド構成案（アウトライン）が生成される
3. 各スライドの要点・話すべきポイントも提案
4. PowerPointに転記して仕上げ
```

#### PDF/レビュー

```
作業フロー:
1. PDFファイルをClaude Codeで読み取り
2. 「要約して」「リスクポイントを洗い出して」と指示
3. 分析結果をMarkdownで出力
4. 比較レビュー: 2つのPDFの差分を抽出
```

## デモシナリオ集

### シナリオ 1: 「30秒で週報ドラフト」

```
手順:
1. 「今週やったこと」を箇条書きで3-5行入力
2. Claude Codeが週報フォーマットに自動変換
3. 課題・来週の予定セクションも自動提案

所要時間: 約30秒
従来の方法: 15-30分
```

### シナリオ 2: 「PDFレポートの即座分析」

```
手順:
1. PDFファイルのパスを伝える
2. Claude Codeが読み取り、3つのポイントで要約
3. 「もっと詳しく」「特定のセクションについて」と深掘り可能

所要時間: 約10秒
従来の方法: 30分-1時間（通読）
```

### シナリオ 3: 「メールのワンコマンド作成」

```
手順:
1. カスタムコマンド `/draft-email` を事前に作成（一度だけ）
2. 以降は `/draft-email 見積もりの件で○○社に返信` と打つだけ
3. 自分の文体で、適切なメールが即座に生成

所要時間: 約5秒
従来の方法: 10-20分
```

### シナリオ 4: 「フォルダの即座理解」

```
手順:
1. Claude Codeを起動するだけ（CLAUDE.mdがあれば）
2. 「このプロジェクトについて教えて」と聞く
3. フォルダ構成、ファイル数、直近の更新状況を即座に把握

所要時間: 約3秒
従来の方法: フォルダを開いて目視確認（数分〜）
```

## カスタムコマンドの魅力

### 一度作れば、ずっと使える

```
/morning          → 毎朝の業務開始準備
/weekly-report    → 週報の自動ドラフト
/draft-email      → メール作成
/meeting-prep     → 会議準備
/review-doc       → 文書レビュー
/translate        → 翻訳
```

### チームで共有できる

```
.claude/commands/ フォルダを共有するだけで、
チーム全員が同じカスタムコマンドを使えます。

「○○さんが作った /weekly-report が便利すぎる」
→ チーム全体の生産性が向上
→ あなたがチームの効率化リーダーに
```

### 育てていける

```
最初: シンプルなテンプレート生成
 ↓ 使いながら改善
中期: 自分の文体・好みを反映
 ↓ さらに高度化
上級: 複数ステップの自動ワークフロー
```
