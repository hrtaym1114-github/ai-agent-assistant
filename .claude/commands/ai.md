# /ai — AIエージェント活用ガイド

## Description

対話形式でユーザーの業務課題を引き出し、最適なAIエージェント活用パターンを提案し、
その場でMVP（最小実装）を構築するガイドセッション。
構築後はカスタムコマンド/スキル化を支援し、成功体験を周囲に広められるところまでサポートする。

## Usage

```
/ai              # フルセッション（発見→提案→体験→定着→展開）
/ai --discover   # 発見フェーズのみ（提案まで）
/ai --build      # 指定パターンのMVP構築のみ
/ai --demo       # Claude Codeの実力デモ
/ai --share      # 成功体験の共有パック作成
```

## Instructions

### 事前準備（黙って読み込む）

以下のファイルを読み込み、セッションで参照できるようにする。内容をユーザーに羅列しない。

1. `docs/patterns.md` — 8パターン定義・事例・定量データ
2. `docs/governance.md` — ガバナンス設定ガイド
3. `docs/showcase.md` — Claude Code実力ショーケース（デモ用）

### 全体フロー

```
Step 0: 開始モード選択   — ガイド型 / 自由記述 / クイック構築 / 実力デモ
Step 1: 発見（Discover）  — 課題を特定
Step 2: 提案（Propose）   — 事例+パターン+ライブデモで「できること」を体感
Step 3: 体験（Experience） — MVPをその場で構築
Step 4: 定着（Embed）     — カスタムコマンド/スキル化で日常に組み込む
Step 5: 成功と展開（Success & Spread） — 成功体験を記録し、周囲に広める
```

## Step Demo: Claude Codeの実力デモ

**目的**: 「これ全部コードなしでできるの？」という驚きを生む。百聞は一見にしかず。

### デモの進め方

1. **現在のフォルダを即座に分析**して結果を見せる:
   - フォルダ構成の可視化
   - ファイル数・種類の集計
   - 直近更新されたファイルの特定
   - 「このフォルダは○○のプロジェクトですね」とコンテキスト理解を示す

2. AskUserQuestion で次のデモを選択:
   - **question**: 「何ができるか実際にお見せします。どれを見てみたいですか？」
   - **header**: "Demo"
   - **options**:
     1. 「ファイル分析＆要約」 — PDFや文書ファイルを読み取って即座に要約します
     2. 「文書の自動生成」 — テンプレートから報告書やメールを一瞬で作ります
     3. 「データの整理・加工」 — CSV/テキストデータを分析・可視化します
     4. 「全部見せて」 — 3つすべてをデモします
   - **multiSelect**: false

### デモ実行

#### デモ A: ファイル分析＆要約

```
実行内容:
1. フォルダ内のドキュメント（PDF、md、txt等）を探す
2. 見つかったファイルを読み取り
3. 内容を構造化して要約
4. 「このファイルのポイントは3つ: ①… ②… ③…」と出力

ポイント: PDFもそのまま読める、画像も認識できることを実演で示す。
もしOffice系ファイル（.docx, .xlsx, .pptx）があった場合:
→ 「Officeファイルもテキスト抽出して分析できます」と紹介
→ Markdown形式での文書生成を実演し、「これをWordに貼り付ければ即完成です」と案内
```

#### デモ B: 文書の自動生成

```
実行内容:
1. ユーザーに「何の文書を作りたいですか？」と1行で聞く
2. 即座にプロフェッショナルな文書を生成
3. 「これを元に修正を指示するだけで、どんどん完成度が上がります」と示す

ポイント: 「ここをもっとフォーマルに」「箇条書きに変えて」等の対話的改善を実演
```

#### デモ C: データの整理・加工

```
実行内容:
1. フォルダ内のCSVやデータファイルを探す
2. データを読み取り、基本統計を出力
3. 簡易なレポートを生成

ポイント: コードが書けなくても、自然言語で「○○を集計して」と言えば動くことを示す
```

### デモ後の誘導

```
いかがでしたか？

今見ていただいたことは、Claude Codeのほんの一部です。
これを毎日の業務に組み込めば、あなただけの効率化が実現できます。

しかも、一度作ったワークフローは「カスタムコマンド」として保存できるので、
明日からは `/daily-start` のようにコマンド一発で実行できます。

あなたの業務に合った設定を一緒に作ってみませんか？
```

→ Step 1（ガイド型）に進む

  3. 「管理部門（人事・総務・法務・経理）」 — バックオフィス業務全般
  4. 「エンジニア・IT」 — 開発、インフラ、DX推進
- **multiSelect**: false

**内部マッピング**（ユーザーには見せない）:

| 回答 | 重点パターン | 提案の切り口 |
|------|------------|------------|
| 営業・マーケ | P4, P7, P8 | 提案書・メール・キャンペーン効率化 |

### Q2: 課題カテゴリの特定

**内部マッピング**:

Q2の回答に応じて異なる選択肢を提示する。AskUserQuestion を使用。

#### Q2 = 情報管理 の場合:
- **question**: 「具体的にはどんな場面で困っていますか？」
- **options**:
  1. 「ファイルがどこにあるか分からなくなる」
  2. 「会議の内容が記録されず流れてしまう」
  3. 「同じ情報を何度も調べ直している」

#### Q2 = 文書作成 の場合:
- **question**: 「どんな文書を書くことが多いですか？」
- **options**:
  1. 「メール（社内・社外）」
  2. 「報告書・レポート・議事録」
  3. 「提案書・企画書」

#### Q2 = 繰り返し の場合:
- **question**: 「一番繰り返している作業は何ですか？」

  2. 「日報・週報・月次レポートの作成」
  3. 「データの集計・整理・加工」

#### Q2 = 安全面 の場合:
- **question**: 「具体的にどんな点が心配ですか？」
- **options**:
  1. 「AIが勝手にファイルを変更・削除しないか」
  2. 「機密情報や個人情報が漏れないか」
  3. 「AIが何をしたか後から説明・確認できるか」

## Step 1-free: 発見（Discover）— 自由記述型

Q0で「自由記述」を選んだ場合のフロー。

### 進め方

1. ユーザーに自由に困りごとを記述してもらう:
   ```
   あなたが仕事で困っていること、面倒に感じていることを
   自由に教えてください。長さは問いません。
   ```

2. ユーザーの記述を分析し、以下を**内部的に**判定する（ユーザーには見せない）:
   - 職種カテゴリ（営業/企画/管理/エンジニア or 不明）
   - 課題カテゴリ（情報管理/文書作成/繰り返し/安全面 or 複合）
   - 具体的な業務シーン

3. 判定結果をもとに、**Step 2（提案）** に直接進む。
   - 「あなたのお話から〜と理解しました」と一言添えてからパターンを提示

4. もし記述が曖昧で判定できない場合のみ、1つだけ補足質問をする。

## Step 2: 提案（Propose）

### パターン選定

Q1（職種）× Q2（課題）の交差から最適パターンを2つ選定する。Q3の回答で提案内容をカスタマイズする。
自由記述型の場合は、記述内容から職種・課題を推定して同じロジックで選定する。

**選定ルール**:
1. Q1の重点パターンとQ2の推奨パターンの**交差**を最優先パターンとする
2. 交差がない場合はQ2の推奨パターンを優先
3. 必ず2パターンを提案する（1つは「おすすめ」マーク付き）

**よくある組み合わせ例**:
| Q1 | Q2 | 提案パターン |
|----|-----|------------|
| 営業 | 文書作成 | P7（おすすめ）+ P4 |
| 営業 | 繰り返し | P8（おすすめ）+ P7 |
| 企画 | 情報管理 | P1（おすすめ）+ P3 |
| 管理部門 | 繰り返し | P8（おすすめ）+ P7 |
| 管理部門 | 安全面 | P5（おすすめ）+ P6 |
| エンジニア | 情報管理 | P1（おすすめ）+ P6 |
| エンジニア | 文書作成 | P7（おすすめ）+ P2 |

### 提案の出力

以下のフォーマットで提案を出力する。**事例と定量データは必ず含める**。

```
## あなたにおすすめの活用方法

回答を踏まえると、次の2つが特に効果的です。

### おすすめ 1: [パターン名]
**所要時間**: [導入時間] | **効果実感**: [時間軸]

[回答を踏まえた1-2文の説明]

> **実際の事例**: [事例を1つ引用]
> [定量データがあれば添える]

### もう1つの選択肢: [パターン名]
**所要時間**: [導入時間] | **効果実感**: [時間軸]

[1-2文の説明]

> **実際の事例**: [事例を1つ引用]
```

## Step 3: 体験（Experience）— MVP構築

選択されたパターンに応じて、対話しながらMVPを構築する。

**共通ルール**:
- 出力先: ユーザーが指定するフォルダ（デフォルトは作業ディレクトリ直下）
- 各ステップでユーザーの入力を待つ（一方的に作らない）
- 完成したら「試しに使ってみましょう」と動作確認を促す
- テンプレートは `templates/` フォルダを参照して生成のベースにする
- **構築完了後は必ず Step 4（定着）に進む**

### P1: 指示書パターン — CLAUDE.md 構築

```
構築フロー:

1. AskUserQuestion: 「このCLAUDE.mdはどんなプロジェクト/チーム用ですか？」
   → プロジェクト概要を記入

2. AskUserQuestion: 「主なフォルダ構成を教えてください。分からなければ作業内容を教えてください。」
   → フォルダ構成を記入（不明なら自動提案）

3. AskUserQuestion: 「AIにやらせたくない操作はありますか？」
   options:
   - 「ファイルの削除は禁止」
   - 「外部への情報送信は確認してほしい」
   - 「特定フォルダ以外は触らないでほしい」
   - 「特に制限なし」
   multiSelect: true

4. → templates/claude-md-basic.md をベースにCLAUDE.md を生成
   → 生成先をユーザーに確認してから書き出し

5. 「できました！試しに『このプロジェクトの概要を教えて』と聞いてみてください。」
   → 動作確認後、Step 4（定着）へ
```

### P4: パーソナライズ学習パターン — 文体学習設定

```
構築フロー:

1. AskUserQuestion: 「どんな文書の文体を学習させたいですか？」
   options:
   - 「メールの文体（挨拶・締め・フレーズ）」
   - 「レポートの構成パターン」
   - 「提案書の論理展開」

2. 「過去の送信メール/文書のサンプルを3-5件見せてください。
    フォルダパスを教えていただくか、テキストを貼り付けてください。」
   → サンプルを分析

3. → スタイルプロファイルをCLAUDE.mdに追記

4. 「学習完了！試しに『○○についてメールを書いて』と依頼してみてください。」
   → 動作確認後、Step 4（定着）へ
```

### P5: ガバナンス・ガードレールパターン — settings.json 構築

```
構築フロー:

1. AskUserQuestion: 「セキュリティレベルを選んでください。」
   options:
   - 「Level 1: 基本（ファイル削除のブロック）」— 個人利用におすすめ
   - 「Level 2: 標準（+ Git破壊操作 + 外部送信ブロック）」— チーム利用におすすめ
   - 「Level 3: 厳格（+ 全ネットワーク通信 + パッケージインストールブロック）」— 機密環境向け

2. AskUserQuestion: 「操作の確認レベルを選んでください。」
   options:
   - 「すべての操作を事前に確認してほしい（plan モード）」— AI初心者におすすめ
   - 「危険な操作だけ確認してほしい（default モード）」
   - 「確認不要、deny設定で安全確保（bypass モード）」— 上級者向け

3. → docs/governance.md を参照して settings.json を生成
   → 生成先をユーザーに確認してから書き出し

4. 「設定完了！試しに『rm test.txt を実行して』と依頼してみてください。
    ブロックされれば成功です。」
```

## --discover モード

Step 0〜2 のみ実行（提案まで）。構築は行わず、レポートに「推奨パターン」を記載して終了。

## --build モード

引数でパターンを指定し、Step 3 のMVP構築のみ実行。
構築後は Step 4（定着）→ Step 5（成功と展開）まで通す。

```
/ai --build P1    # P1: 指示書パターンの構築
/ai --build P5    # P5: ガバナンスパターンの構築
/ai --build P7    # P7: テンプレート駆動パターンの構築
/ai --build P8    # P8: ルーティン自動化パターンの構築
```

## --demo モード

Step Demo のみ実行。Claude Codeの実力をデモする。
デモ後は「もっと深く試してみませんか？」と /ai の実行を案内。

## --share モード

Step 5-3 の展開支援のみ実行。既に成功体験がある場合に共有パックを作成する。
過去のセッションレポート（sessions/ 内）を参照して共有資料を生成。

---

## 注意事項

- **質問は1回に1つ**: AskUserQuestionは毎回1問ずつ。ユーザーの回答を待ってから次に進む
- **押し付けない**: 「やってみませんか？」であって「やりましょう」ではない
- **事例は具体的に**: 「ある企業では〜」ではなく「デンソーでは〜」「住友商事では〜」と企業名を出す
- **定量データを添える**: 「効率化できます」ではなく「月12時間の削減」「3倍に増加」等
- **ファイル生成時は確認**: 生成先のパスをユーザーに確認してから書き出す
- **成功体験を大切に**: 小さくても「できた！」を感じてもらう。動作確認は必ず行う
- **スキル化を自然に提案**: 構築後に「コマンド化しませんか？」の流れを忘れない
- **Office連携に触れる**: 文書系の課題では必ずPDF読み取り・Markdown生成等の能力を紹介する
- **展開を後押し**: 「周りにも広めませんか？」は押し付けでなく選択肢として提示する
