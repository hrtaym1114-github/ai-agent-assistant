# /ai-adoption-guide — AIエージェント活用ガイド

## Description

対話形式でユーザーの業務課題を引き出し、最適なAIエージェント活用パターンを提案し、その場でMVP（最小実装）を構築するガイドセッション。

## Usage

```
/ai-adoption-guide              # フルセッション（発見→提案→体験）
/ai-adoption-guide --discover   # 発見フェーズのみ（提案まで）
/ai-adoption-guide --build      # 指定パターンのMVP構築のみ
```

## Instructions

### 事前準備（黙って読み込む）

以下のファイルを読み込み、セッションで参照できるようにする。内容をユーザーに羅列しない。

1. `docs/patterns.md` — 8パターン定義・事例・定量データ
2. `docs/governance.md` — ガバナンス設定ガイド

### 全体フロー

```
Step 0: 開始モード選択  — ガイド型 or 自由記述 or クイック構築
Step 1: 発見（Discover）  — 課題を特定
Step 2: 提案（Propose）   — 事例+パターンで「できること」を提示
Step 3: 体験（Experience） — MVPをその場で構築
Step 4: まとめ（Report）  — やったことと次のアクションを出力
```

---

## Step 0: 開始モード選択

### 導入メッセージ

以下の趣旨で、フレンドリーに挨拶する（一字一句同じでなくてよい）:

```
AIエージェント活用ガイドへようこそ。

あなたの業務課題に合ったAI活用方法を一緒に見つけて、
その場で設定まで作れるガイドセッションです。
```

### Q0: 開始モードの確認

**初めての方も、リピーターも快適に使えるよう、まず進め方を選んでもらう。**

AskUserQuestion を使用:

- **question**: 「どのように始めますか？」
- **header**: "Start Mode"
- **options**:
  1. 「ガイド型（おすすめ）」 — 3つの質問で最適なパターンを一緒に見つけます。初めての方向け
  2. 「自由記述」 — 困っていることを自分の言葉で伝えてください。直接パターンを提案します
  3. 「クイック構築」 — 作りたいものが決まっている方。P1/P5/P7/P8のMVPを直接構築します
- **multiSelect**: false

**分岐ロジック**:
| 選択 | 遷移先 |
|------|--------|
| ガイド型 | → Step 1（Q1から開始） |
| 自由記述 | → Step 1-free（自由記述フロー） |
| クイック構築 | → Step 3 直接（パターン選択→構築） |

## Step 1: 発見（Discover）— ガイド型

### Q1: 職種の確認

AskUserQuestion を使用:

- **question**: 「あなたのお仕事について教えてください。一番近いものはどれですか？」
- **header**: "Job Role"
- **options**:
  1. 「営業・マーケティング」 — 顧客対応、提案書、キャンペーン等
  2. 「企画・事業開発」 — 戦略立案、リサーチ、プロジェクト推進
  3. 「管理部門（人事・総務・法務・経理）」 — バックオフィス業務全般
  4. 「エンジニア・IT」 — 開発、インフラ、DX推進
- **multiSelect**: false

**内部マッピング**（ユーザーには見せない）:

| 回答 | 重点パターン | 提案の切り口 |
|------|------------|------------|
| 営業・マーケ | P4, P7, P8 | 提案書・メール・キャンペーン効率化 |
| 企画・事業開発 | P1, P3, P7 | 情報整理とアイデアの構造化 |
| 管理部門 | P5, P7, P8 | 定型業務の自動化とガバナンス |
| エンジニア・IT | P1, P2, P6 | 開発プロセスと知識の蓄積 |

### Q2: 課題カテゴリの特定

AskUserQuestion を使用:

- **question**: 「仕事の中で一番『面倒だな』『時間がかかるな』と感じることは？」
- **header**: "Challenge"
- **options**:
  1. 「情報やファイルの管理が大変」 — どこに何があるかわからない、探す時間が長い
  2. 「文書・メール・資料の作成に時間がかかる」 — 毎回ゼロから書いている
  3. 「毎日同じ作業の繰り返しがある」 — ルーティンワークに時間を取られる
  4. 「AIを使いたいけど安全面やルールが不安」 — セキュリティ、ガバナンス
- **multiSelect**: false

**内部マッピング**:

| 回答 | 課題カテゴリ | 推奨パターン |
|------|------------|------------|
| 情報管理 | information | P1, P3 |
| 文書作成 | document | P4, P7 |
| 繰り返し | routine | P7, P8 |
| 安全面 | security | P5, P6 |

### Q3: 具体的な業務シーンの深掘り

Q2の回答に応じて異なる選択肢を提示する。AskUserQuestion を使用。

#### Q2 = 情報管理 の場合:
- **question**: 「具体的にはどんな場面で困っていますか？」
- **options**:
  1. 「ファイルがどこにあるか分からなくなる」
  2. 「会議の内容が記録されず流れてしまう」
  3. 「同じ情報を何度も調べ直している」

#### Q2 = 文書作成 の場合:
- **question**: 「どんな文書を書くことが多いですか？」
- **options**:
  1. 「メール（社内・社外）」
  2. 「報告書・レポート・議事録」
  3. 「提案書・企画書」

#### Q2 = 繰り返し の場合:
- **question**: 「一番繰り返している作業は何ですか？」
- **options**:
  1. 「朝のメール確認や日次の準備作業」
  2. 「日報・週報・月次レポートの作成」
  3. 「データの集計・整理・加工」

#### Q2 = 安全面 の場合:
- **question**: 「具体的にどんな点が心配ですか？」
- **options**:
  1. 「AIが勝手にファイルを変更・削除しないか」
  2. 「機密情報や個人情報が漏れないか」
  3. 「AIが何をしたか後から説明・確認できるか」

## Step 1-free: 発見（Discover）— 自由記述型

Q0で「自由記述」を選んだ場合のフロー。

### 進め方

1. ユーザーに自由に困りごとを記述してもらう:
   ```
   あなたが仕事で困っていること、面倒に感じていることを
   自由に教えてください。長さは問いません。
   ```

2. ユーザーの記述を分析し、以下を**内部的に**判定する（ユーザーには見せない）:
   - 職種カテゴリ（営業/企画/管理/エンジニア or 不明）
   - 課題カテゴリ（情報管理/文書作成/繰り返し/安全面 or 複合）
   - 具体的な業務シーン

3. 判定結果をもとに、**Step 2（提案）** に直接進む。
   - 「あなたのお話から〜と理解しました」と一言添えてからパターンを提示

4. もし記述が曖昧で判定できない場合のみ、1つだけ補足質問をする。

## Step 2: 提案（Propose）

### パターン選定

Q1（職種）× Q2（課題）の交差から最適パターンを2つ選定する。Q3の回答で提案内容をカスタマイズする。
自由記述型の場合は、記述内容から職種・課題を推定して同じロジックで選定する。

**選定ルール**:
1. Q1の重点パターンとQ2の推奨パターンの**交差**を最優先パターンとする
2. 交差がない場合はQ2の推奨パターンを優先
3. 必ず2パターンを提案する（1つは「おすすめ」マーク付き）

**よくある組み合わせ例**:
| Q1 | Q2 | 提案パターン |
|----|-----|------------|
| 営業 | 文書作成 | P7（おすすめ）+ P4 |
| 営業 | 繰り返し | P8（おすすめ）+ P7 |
| 企画 | 情報管理 | P1（おすすめ）+ P3 |
| 管理部門 | 繰り返し | P8（おすすめ）+ P7 |
| 管理部門 | 安全面 | P5（おすすめ）+ P6 |
| エンジニア | 情報管理 | P1（おすすめ）+ P6 |
| エンジニア | 文書作成 | P7（おすすめ）+ P2 |

### 提案の出力

以下のフォーマットで提案を出力する。**事例と定量データは必ず含める**。

```
## あなたにおすすめの活用方法

回答を踏まえると、次の2つが特に効果的です。

### おすすめ 1: [パターン名]
**所要時間**: [導入時間] | **効果実感**: [時間軸]

[回答を踏まえた1-2文の説明]

> **実際の事例**: [事例を1つ引用]
> [定量データがあれば添える]

### もう1つの選択肢: [パターン名]
**所要時間**: [導入時間] | **効果実感**: [時間軸]

[1-2文の説明]

> **実際の事例**: [事例を1つ引用]
```

### 体験への誘導

提案後、AskUserQuestion で構築に進むか確認:

- **question**: 「今から一緒に設定を作ってみませんか？15-30分で最初の一歩が踏み出せます。」
- **header**: "Next Step"
- **options**:
  1. 「[おすすめパターン名]から始める」 — [所要時間]
  2. 「[もう1つのパターン名]から始める」 — [所要時間]
  3. 「もう少し詳しく聞きたい」
  4. 「今日は情報収集だけにする」
- **multiSelect**: false

**「もう少し詳しく」を選んだ場合**: 追加の事例を2-3件提示し、再度構築に誘導する。

**「情報収集だけ」を選んだ場合**: Step 4（レポート出力）に進む。

## Step 3: 体験（Experience）— MVP構築

選択されたパターンに応じて、対話しながらMVPを構築する。

**共通ルール**:
- 出力先: ユーザーが指定するフォルダ（デフォルトは作業ディレクトリ直下）
- 各ステップでユーザーの入力を待つ（一方的に作らない）
- 完成したら「試しに使ってみましょう」と動作確認を促す
- テンプレートは `templates/` フォルダを参照して生成のベースにする

### P1: 指示書パターン — CLAUDE.md 構築

```
構築フロー:

1. AskUserQuestion: 「このCLAUDE.mdはどんなプロジェクト/チーム用ですか？」
   → プロジェクト概要を記入

2. AskUserQuestion: 「主なフォルダ構成を教えてください。分からなければ作業内容を教えてください。」
   → フォルダ構成を記入（不明なら自動提案）

3. AskUserQuestion: 「AIにやらせたくない操作はありますか？」
   options:
   - 「ファイルの削除は禁止」
   - 「外部への情報送信は確認してほしい」
   - 「特定フォルダ以外は触らないでほしい」
   - 「特に制限なし」
   multiSelect: true

4. → templates/claude-md-basic.md をベースにCLAUDE.md を生成
   → 生成先をユーザーに確認してから書き出し

5. 「できました！試しに『このプロジェクトの概要を教えて』と聞いてみてください。」
```

### P4: パーソナライズ学習パターン — 文体学習設定

```
構築フロー:

1. AskUserQuestion: 「どんな文書の文体を学習させたいですか？」
   options:
   - 「メールの文体（挨拶・締め・フレーズ）」
   - 「レポートの構成パターン」
   - 「提案書の論理展開」

2. 「過去の送信メール/文書のサンプルを3-5件見せてください。
    フォルダパスを教えていただくか、テキストを貼り付けてください。」
   → サンプルを分析

3. → スタイルプロファイルをCLAUDE.mdに追記

4. 「学習完了！試しに『○○についてメールを書いて』と依頼してみてください。」
```

### P5: ガバナンス・ガードレールパターン — settings.json 構築

```
構築フロー:

1. AskUserQuestion: 「セキュリティレベルを選んでください。」
   options:
   - 「Level 1: 基本（ファイル削除のブロック）」— 個人利用におすすめ
   - 「Level 2: 標準（+ Git破壊操作 + 外部送信ブロック）」— チーム利用におすすめ
   - 「Level 3: 厳格（+ 全ネットワーク通信 + パッケージインストールブロック）」— 機密環境向け

2. AskUserQuestion: 「操作の確認レベルを選んでください。」
   options:
   - 「すべての操作を事前に確認してほしい（plan モード）」— AI初心者におすすめ
   - 「危険な操作だけ確認してほしい（default モード）」
   - 「確認不要、deny設定で安全確保（bypass モード）」— 上級者向け

3. → docs/governance.md を参照して settings.json を生成
   → 生成先をユーザーに確認してから書き出し

4. 「設定完了！試しに『rm test.txt を実行して』と依頼してみてください。
    ブロックされれば成功です。」
```

### P7: テンプレート駆動パターン — テンプレート作成

```
構築フロー:

1. AskUserQuestion: 「一番よく作る文書の種類は何ですか？」
   options:
   - 「メール（定型返信、初回提案、フォローアップ等）」
   - 「報告書（日報、週報、月次レポート等）」
   - 「企画書・提案書」
   - 「議事録・会議まとめ」

2. 「いつもその文書に含める項目を教えてください。
    箇条書きで大丈夫です。分からなければ最近作ったものを見せてください。」
   → 構造を把握

3. → templates/ 内の該当テンプレートをベースにカスタムテンプレートを生成
   → 生成先をユーザーに確認してから書き出し

4. 「テンプレートができました！
    試しに『このテンプレートを使って○○のドラフトを作って』と依頼してみてください。」
```

### P8: ルーティン自動化パターン — Skill 作成

```
構築フロー:

1. AskUserQuestion: 「自動化したいルーティン作業は何ですか？」
   options:
   - 「朝の業務開始準備（予定確認、TODO整理）」
   - 「日報・週報の自動ドラフト」
   - 「データの定期集計・レポート」
   - 「メールの整理・優先度付け」

2. 「その作業の具体的な手順を教えてください。
    『毎朝やること: 1. ○○ 2. △△ 3. □□』のような形で。」
   → 手順を把握

3. AskUserQuestion: 「出来上がりはどんな形式がいいですか？」
   options:
   - 「Markdownファイル（ノート、レポート等）」
   - 「TODO リスト」
   - 「テキスト出力（画面表示のみ）」

4. → templates/daily-start-skill.md をベースに .claude/commands/[名前].md を生成
   → 生成先をユーザーに確認してから書き出し

5. 「Skillができました！
    次回から `/[コマンド名]` と打つだけで自動実行されます。
    試しに今実行してみましょう。」
```

## Step 4: まとめ（Report）

### レポート出力

セッション完了時に以下のレポートを生成する。

**出力先**: `sessions/` フォルダ（なければ作成）

**ファイル名**: `YYYYMMDD_adoption-session_[ユーザー識別].md`

**テンプレート**:

```markdown
# AIエージェント活用セットアップレポート

**日付**: [今日の日付]
**対象者**: [ユーザー名 or 匿名]

## あなたのプロフィール
- **職種**: [Q1の回答]
- **課題**: [Q2の回答]
- **具体的な場面**: [Q3の回答]

## 提案されたパターン
1. **[パターン名]**: [1文の説明]
2. **[パターン名]**: [1文の説明]

## 構築したもの
- **パターン**: [選択されたパターン]
- **生成ファイル**: [ファイルパス一覧]
- **動作確認**: [OK / 未確認]

## 次のステップ
1. **今週**: 構築したものを日常業務で使ってみる
2. **来週**: 効果を振り返り、微調整する
3. **次の導入候補**: [未選択のパターン名]（[導入時間]で追加可能）

## 参考にした事例
- [提案時に使った事例名とソース]

---
*Generated by /ai-adoption-guide*
```

```
お疲れさまでした！
最初の一歩が完了しました。

まずは1週間、今日作ったものを使ってみてください。
困ったことがあれば `/ai-adoption-guide` をもう一度実行すれば、
続きからサポートします。

周りの方にも紹介したい場合は、このレポートを共有してください。
```

## --discover モード

Step 0〜2 のみ実行（提案まで）。構築は行わず、レポートに「推奨パターン」を記載して終了。

## --build モード

引数でパターンを指定し、Step 3 のMVP構築のみ実行。

```
/ai-adoption-guide --build P1    # P1: 指示書パターンの構築
/ai-adoption-guide --build P5    # P5: ガバナンスパターンの構築
/ai-adoption-guide --build P7    # P7: テンプレート駆動パターンの構築
/ai-adoption-guide --build P8    # P8: ルーティン自動化パターンの構築
```

---

## 注意事項

- **質問は1回に1つ**: AskUserQuestionは毎回1問ずつ。ユーザーの回答を待ってから次に進む
- **押し付けない**: 「やってみませんか？」であって「やりましょう」ではない
- **事例は具体的に**: 「ある企業では〜」ではなく「デンソーでは〜」「住友商事では〜」と企業名を出す
- **定量データを添える**: 「効率化できます」ではなく「月12時間の削減」「3倍に増加」等
- **ファイル生成時は確認**: 生成先のパスをユーザーに確認してから書き出す
