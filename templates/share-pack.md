# チーム展開パッケージ テンプレート

> 成功体験をチームに展開するための配布パッケージ。
> `/ai --share` で「共有パックを作ってほしい」を選ぶと自動生成されます。

---

## 共有パックの構成

```
share-pack/
├── README.md               # 紹介資料（なぜ使うべきか）
├── quick-start.md          # 5分で始めるセットアップ手順
├── success-story.md        # あなたの成功体験
└── try-it/                 # すぐ試せるサンプル一式
    ├── CLAUDE.md           # 調整済みの指示書
    └── .claude/
        ├── commands/       # 作成したカスタムコマンド
        │   └── [skill].md
        └── settings.json   # セキュリティ設定
```

---

## 各ファイルのテンプレート

### README.md — 紹介資料

```markdown
# Claude Code で業務を効率化しよう

## [あなたの名前]からの紹介

[あなたの名前]です。
Claude Codeを使って[具体的な課題]を解決しました。

### 実際の効果

- **Before**: [課題の具体的状況]
- **After**: [改善された状態]
- **効果**: [定量データ]

### あなたも5分で始められます

1. Claude Codeを起動
2. この `try-it/` フォルダに移動
3. `/ai` と入力 → ガイドが始まります

### 私が作ったカスタムコマンド

| コマンド | できること |
|---------|----------|
| `/[名前]` | [説明] |

これらは `try-it/` フォルダに入っているので、すぐ使えます。

### もっと知りたい方へ

- 詳しい成功体験: `success-story.md` を読んでください
- セットアップ手順: `quick-start.md` を参照
- 安全性について: セキュリティ設定はLevel 2（標準）で安全です
```

### quick-start.md — 5分セットアップ

```markdown
# 5分で始めるClaude Code

## 前提条件

- Claude Codeがインストール済み
  （まだの方は管理者に相談してください）

## Step 1: フォルダをコピー（1分）

この `try-it/` フォルダの中身を、あなたの作業フォルダにコピーしてください。

コピーするもの:
- `CLAUDE.md` → 作業フォルダ直下に配置
- `.claude/` フォルダ → 作業フォルダ直下に配置

## Step 2: Claude Codeを起動（30秒）

作業フォルダで Claude Code を起動します:

## Step 3: 動作確認（1分）

以下を試してみてください:

1. 「このプロジェクトの概要を教えて」 → CLAUDE.mdの内容を理解しているか確認
2. 「/[カスタムコマンド名]」 → コマンドが動作するか確認

## Step 4: 自分専用にカスタマイズ（2-3分）

自分の業務に合わせてカスタマイズしたい場合:

```
/ai
```

と入力すると、対話ガイドが始まります。
質問に答えるだけで、あなた専用の設定が構築されます。

## 困ったら

- `/ai` をもう一度実行すれば、いつでもガイドが始まります
- CLAUDE.md を自由に編集して、AIへの指示を調整できます
- .claude/commands/ にファイルを追加すれば、カスタムコマンドを増やせます
```

## 配布方法

### 方法 1: フォルダ共有（最も簡単）

```
1. share-pack/ フォルダをZIP圧縮
2. Teams/メール/共有フォルダで配布
3. 受け取った人は展開して try-it/ をコピーするだけ
```

### 方法 2: Teams/Slack で紹介

```
投稿テンプレート:

---
Claude Codeで[作業名]を効率化しました。

Before: [時間] → After: [時間]（[削減率]%削減）

試してみたい方に設定一式を共有します。
5分でセットアップできます。

添付: share-pack.zip
---
```

### 方法 3: チーム会議でデモ

```
3分デモのアジェンダ:

1. 課題の共有（30秒）
   → 「毎週○○に△分かかっていました」

2. ライブデモ（1分30秒）
   → 実際に `/[コマンド名]` を実行して見せる
   → Before/After を目の前で実演

3. 始め方の案内（1分）
   → 「この共有パックを展開するだけ」
   → 「/ai と打てば対話ガイドが始まります」
```

## 展開のコツ

### 最初は1人に集中

- いきなりチーム全体ではなく、興味のある1人に紹介
- その人が成功すれば、自然に広まる

### 「自分にもできそう」を大事に

- 「コード不要」を強調
- 「質問に答えるだけ」を強調
- 実際に目の前でデモする

### 小さく始める

- 最初は1つのカスタムコマンドから
- 「毎日使う1つの作業」を自動化するだけで十分
- 成功体験が次の導入を自然に促す
