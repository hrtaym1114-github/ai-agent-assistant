# 朝ルーティン Skill テンプレート

> このテンプレートをベースに、自分の朝ルーティンに合わせた Skill を作成できます。
> 完成したら `.claude/commands/` フォルダに配置し、`/daily-start` で実行。

---

## テンプレート

以下を `.claude/commands/daily-start.md` として保存:

```markdown
# /daily-start — 朝の業務開始ルーティン

## Instructions

今日の日付で業務開始の準備を行ってください。

### Step 1: 今日の日付と曜日を確認
- `date` コマンドで正確な日付・曜日を取得

### Step 2: 今日のタスクリスト作成
以下の情報をもとに、今日のタスクリストを作成:
- [昨日の未完了タスクがあれば引き継ぎ]
- [定例のルーティン（例: 毎週月曜は会議資料準備）]
- [カレンダーの予定（確認できる場合）]

### Step 3: Top 3 Priorities を決定
タスクリストから最も重要な3つを選び、優先順位をつける

### Step 4: 出力
以下のフォーマットで出力:

```
# [YYYY-MM-DD]（[曜日]）

## Top 3 Priorities
1. [ ] [最重要タスク]
2. [ ] [2番目に重要]
3. [ ] [3番目に重要]

## Today's Tasks
- [ ] [タスク1]
- [ ] [タスク2]
- [ ] [タスク3]
- [ ] [タスク4]

## Schedule
- [HH:MM] [予定1]
- [HH:MM] [予定2]

## Notes
- [メモ・注意事項]
```
```

---

## カスタマイズ例

### 営業職向け

```markdown
### Step 2 に追加:
- 今日訪問する顧客リストを確認
- 前回の商談メモから準備事項を抽出
- 提出期限が近い見積もりを確認
```

### 管理職向け

```markdown
### Step 2 に追加:
- チームメンバーの昨日の日報を要約
- 未承認の申請・稟議を確認
- 今週の部門KPI進捗を確認
```

### エンジニア向け

```markdown
### Step 2 に追加:
- 未レビューのPR一覧を確認（gh pr list）
- 自分にアサインされたIssueを確認
- CI/CDの状態を確認
```

## 他のルーティン Skill の例

| Skill名 | トリガー | 内容 |
|---------|---------|------|
| `/daily-end` | 終業時 | 今日の成果まとめ、明日への引き継ぎ |
| `/weekly-review` | 金曜午後 | 今週の振り返り、来週の計画 |
| `/meeting-prep [議題]` | 会議前 | 議題の整理、論点・質問リスト作成 |
| `/inbox-zero` | 随時 | 未読メール/未処理タスクの整理 |
