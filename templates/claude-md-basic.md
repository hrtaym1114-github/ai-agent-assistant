# CLAUDE.md テンプレート（基本版）

> このテンプレートをコピーして、自分のプロジェクトに合わせてカスタマイズしてください。
> `[...]` の部分を記入します。

---

```markdown
# CLAUDE.md

## このプロジェクトについて

[1-2文でプロジェクトの概要を記述]
[例: 営業チームの提案書・見積書を管理するフォルダです。顧客ごとにフォルダが分かれています。]

## フォルダ構成

[主要なフォルダとその役割を記述]

```
project/
├── [フォルダ名]/    # [役割の説明]
├── [フォルダ名]/    # [役割の説明]
├── [フォルダ名]/    # [役割の説明]
└── templates/       # テンプレート置き場
```

## 命名規則

- ファイル名: [例: YYYY-MM-DD_顧客名_種別.md]
- フォルダ名: [例: 英語、ハイフン区切り]

## ルール

- [例: ファイルの削除は行わない]
- [例: 外部サービスへの情報送信前は必ず確認する]
- [例: templates/ フォルダ内のファイルは編集しない（コピーして使う）]

## よく使うタスク

- [例: 「新しい提案書を作って」→ templates/ をベースに生成]
- [例: 「今週の活動をまとめて」→ 各フォルダの更新を集計してレポート作成]
- [例: 「○○のファイルを探して」→ フォルダ構成に基づいて検索]
```

---

## カスタマイズのヒント

### 最低限入れるべき3要素

1. **プロジェクト概要** — AIが「何のための仕事か」を理解する
2. **フォルダ構成** — AIが「どこに何があるか」を理解する
3. **禁止事項** — AIがやってはいけないことを明確にする

### 応用: 常駐アシスタント機能を追加（P2）

CLAUDE.mdに以下を追加すると、すべての応答に自動処理が付加されます:

```markdown
## 常駐アシスタント

すべての応答に対して、以下の処理を自動的に付加すること:
- [例: 日本語の入力に対しては英訳も表示する]
- [例: 文書作成時はSEO観点のチェックポイントも表示する]
```
