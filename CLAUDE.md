# AI Agent Assistant

## Overview

このプロジェクトは、AIエージェント（Claude Code等）を業務に活用するためのオンボーディングキットです。
対話型ガイドであなたの業務課題を特定し、最適な活用パターンを提案、その場で設定を構築します。

## Quick Start

1. `/ai` を実行 — 対話ガイドが始まります
2. 質問に答えるだけで、あなたに合った活用方法を提案
3. その場でMVP（最小実装）を構築

## Available Commands

| コマンド | 説明 |
|---------|------|
| `/ai` | フルガイドセッション（発見→提案→体験） |
| `/ai --discover` | 提案まで（情報収集のみ） |
| `/ai --build P1` | CLAUDE.md構築 |
| `/ai --build P5` | ガバナンス設定構築 |
| `/ai --build P7` | テンプレート構築 |
| `/ai --build P8` | ルーティン自動化構築 |

## 8つの活用パターン

詳細は `docs/patterns.md` を参照。

| # | パターン | 解決する課題 | 導入時間 |
|---|---------|-------------|---------|
| P1 | プロジェクト指示書 | AIに毎回同じ説明をしている | 30分 |
| P2 | 常駐アシスタント | スキルを日常業務で伸ばしたい | 15分 |
| P3 | 情報トリアージ | 情報が散らかって見つからない | 1-2時間 |
| P4 | パーソナライズ学習 | AIの文章が自分っぽくない | 2-3時間 |
| P5 | ガバナンス | AIの暴走やセキュリティが心配 | 30分 |
| P6 | 知識資産記録 | AIとのやり取りが消えてしまう | 1-2時間 |
| P7 | テンプレート駆動 | 毎回ゼロから文書を書いている | 1時間 |
| P8 | ルーティン自動化 | 毎日同じ作業を手動でやっている | 1-2時間 |

## Safety

このキットにはLevel 2（標準）のセキュリティ設定が含まれています:
- ファイル削除をブロック
- Git破壊操作をブロック
- 外部サービスへのPOST/PUT/DELETE送信をブロック

カスタマイズ方法は `docs/governance.md` を参照。

## Rules

- ファイル生成時は必ずユーザーに出力先を確認してから書き出す
- 既存ファイルの大幅な書き換えは事前に内容を説明し確認を取る
- 機密情報（個人名、メールアドレス、顧客情報等）が含まれるファイルはマスキングして扱う
- 作業範囲はユーザーが指定したフォルダ内に限定する
- ファイルの削除は行わない
